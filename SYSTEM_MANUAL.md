# TuneBoy System Manual & Logic Analysis

## 1. Randomization Parameters
Every time the **Randomize** function is triggered, the system generates a new "Seed State" by selecting values for the following parameters. These discrete options define the overall "genre" and texture of the session.

| Parameter | Range/Options | Manual Control? | Description | Count |
| :--- | :--- | :--- | :--- | :--- |
| **⏱️ Tempo** | 40–180 BPM | ✅ Pot 0 | Speed of the clock. | 4096 |
| **🎹 Transpose** | -12 to +12 | ✅ Pot 1 | Global pitch shift in semitones. | 25 |
| **🎼 Scale Mode** | Major / Minor | ✅ DIP 1 | Determines harmonic ruleset (Chord Qualities, Scales). | 2 |
| **🔄 Chord Mode** | Preset/Procedural | ✅ DIP 2 | Toggles between fixed progression or random walk. | 2 |
| **🐢 Half Time** | On/Off | ✅ DIP 3 | Toggles half-speed metric modulation. | 2 |
| **🎛️ Osc Mutes** | On/Off | ✅ DIP 4-7 | Manually mute Bass, Mid, Melody, or Noise. | 16 |
| **🎛️ Oscillator Mode** | 1–6 | ❌ | Sets active instrument combo (e.g. "Bass+Lead"). | 6 |
| **〰️ Arpeggio Mode** | 1–5 | ❌ | Pattern logic for Mid voice (Up, Down, etc.). | 5 |
| **🎸 Bass Mode** | 1–3 | ❌ | Bassline logic (Root, Octaves, Fifth). | 3 |
| **🥁 Drum Mode** | 1–16 | ❌ | Selects drum pattern. | 16 |
| **🔄 Chord Prog** | 1–50 | ❌ | Selects harmonic progression. | 50 |

## 2. Procedural Engines
Beyond the static settings above, the system uses several active procedural engines that generate music in real-time. This means that even with the *exact same* randomization settings, the music will never repeat itself exactly.

### A. The "Smart" Melody Engine
The melody is not a loop; it is a **Markov-Chain-style Random Walk** with intelligent constraints:
*   **Phrase Generation**: Generates a random rhythmic motif (e.g., Gallop, 16ths).
*   **Lookahead Voice Leading**: Before picking a note, it looks at the *future* chord to find "Common Tones" (notes shared between current and next scales) to bridge the gap.
*   **Call & Response**: It generates a "Call" phrase, then repeats it as a "Response" with slight mutations (pitch shifts, octave jumps).
*   **Quantization**: Every note is mathematically snapped to the current chord's scale to ensure no dissonant clashes.

### B. The Chord Logic
*   **Predefined Mode**: Cycles through one of 50 expert-written progressions (e.g., I-IV-I-IV).
*   **Procedural Mode**: Uses a probabilistic transition table (Markov Chain). For example, if currently on a **V** chord, there might be a 90% chance to resolve to **I** and a 10% chance to deceive to **vi**.

### C. The Rhythm Engine
*   **Melody**: Uses a "Rhythm Motif" system. It picks a rhythmic seed (e.g., `[Step, Step, Skip]`) and applies it to the entire musical phrase.
*   **Drums**: Uses a 16-step sequencer with variable drum samples generated by filtered White Noise.

## 3. Combinatorial Calculation
How many unique "Songs" can TuneBoy generate?

### Discrete "Seed" Combinations
Calculating only the parameters controlled by the Randomize button (excluding manual overrides like Tempo/Transpose):

*   **Scale Mode**: 2 options
*   **Oscillator Mode**: 6 options
*   **Arpeggio Mode**: 5 options
*   **Bass Mode**: 3 options
*   **Drum Mode**: 16 options
*   **Chord Progressions**: 50 options

$$
2 \times 6 \times 5 \times 3 \times 16 \times 50 = \mathbf{144,000}
$$

**Result**: There are over **144,000** unique "randomized seeds" available at the press of a button.
*Note: This number explodes into the millions if you consider the manual permutations of Tempo, Transpose, and Mutes.*

### Total Musical Possibilities
Since the **Melody** and **Rhythm** are procedurally generated in real-time and technically infinite (never looping exactly the same way twice), the actual number of possible audio output streams is **Infinite**.

Even if we locked the melody to a 16-step phrase with 2 octaves of range:
*   $12 \text{ notes} \times 2 \text{ octaves} = 24 \text{ options per step}$
*   $24^{16} \approx 1.3 \times 10^{22}$ unique melodies *per phrase*.

**Conclusion**: You will likely never hear the exact same minute of music twice.

## 4. Connectivity & MIDI
TuneBoy is more than just a self-contained synth; it can drive your entire studio.

*   **MIDI Out**: Emits standard MIDI Note On/Off messages via `Serial1` (TX Pin).
*   **Baud Rate**: 31250 bps (Standard MIDI).
*   **Channel Mapping**:
    *   **Channel 1**: Bass (Anchors the progression).
    *   **Channel 2**: Arpeggiator (Provides motion).
    *   **Channel 3**: Melody (The "Smart" Lead).
    *   *(Channel 10/Drums are internal only via Noise channel).*
